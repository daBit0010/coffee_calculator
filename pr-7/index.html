Starter para GitHub Pages

Un punto de partida moderno para publicar un micrositio estático sin pipelines ni build. Copia este HTML, personaliza textos, estilos y assets, y publícalo desde cualquier repositorio.

Crear sitio en GitHub Pages Ver repositorio de ejemplo

Qué incluye

HTML

Estructura accesible

Secciones básicas (hero, beneficios, pasos y contacto) listas para editar. Agrega componentes propios sin romper el layout.

CSS

Estilos sin dependencias

Todo el estilo vive inline para evitar builds. Ajusta colores, tipografías y espaciados desde las variables CSS del inicio.

DX

Sin configuraciones complejas

No requiere Node, bundlers ni frameworks. Abre el archivo en tu navegador y verás exactamente lo que se publica.

Cómo publicar en minutos

  1. Clona o copia el starter Usa la opción de plantilla, submódulo o un simple sync-site.sh para traer los archivos a tu proyecto.
  2. Personaliza contenido y estilos Edita títulos, textos, enlaces y agrega tus imágenes. Opcional: añade secciones nuevas o integra analytics.
  3. Activa GitHub Pages En Settings → Pages publica desde el branch principal y listo. Tu sitio quedará disponible en cuestión de segundos.
¿Necesitas contenido dinámico? Puedes enlazar APIs públicas con JavaScript o agregar un flujo de compilación más adelante.
